# 카드 게임 모음

`index.html` 은 이제 **게임 허브(시작 페이지)** 입니다. 여기서 원카드 · 도둑잡기 · 블랙잭 · 솔리테어를 고를 수 있어요.

## 배포 / 업로드 안내 (중요)
- repo 에 **모든 파일을 같은 폴더(루트)에 함께** 올리세요. 새로 추가된 파일: `index.html`(허브), `onecard.html`(원카드 본체), `shared.css`, `shared.js`, `oldmaid.html`, `oldmaid.js`, `blackjack.html`, `blackjack.js`, `solitaire.html`, `solitaire.js`.
- 기존 원카드 파일(`game.js` `app.js` `effects.js` `realtime.js` `style.css` `firebase-config.js`)은 **그대로 둡니다.** 원카드는 이제 `index.html` 이 아니라 `onecard.html` 에서 돌아갑니다(허브에서 클릭).
- ⚠️ **`firebase-config.js` 는 절대 빈 버전으로 덮어쓰지 마세요.** 실제 값이 들어간 현재 파일을 유지해야 온라인 멀티가 켜집니다.
- 업로드 후 강력 새로고침(Ctrl+Shift+R).

## 게임별 상태
- **원카드** — 패스앤플레이 + 온라인 멀티 (완성)
- **도둑잡기** — 싱글(봇) + **온라인 멀티 완성** (원카드와 같은 Firebase 사용, 2~5명)
- **블랙잭** — 싱글(딜러) 완성 · 멀티 예정
- **솔리테어** — 싱글 완성(탭으로 이동, 되돌리기, 자동완성) · 경쟁전 예정

새 게임들은 `shared.css` / `shared.js`(카드 렌더·사운드·이펙트 공용)를 쓰고, 효과음·모바일 비율 자동조정·다크 테이블 톤을 원카드와 똑같이 공유합니다.

# 원카드 · 멀티 (이하 원카드 전용 안내)

마스터가 판을 세팅하고 시작하는 원카드 게임. **한 기기에서 돌아가며(패스앤플레이)** 바로 칠 수도 있고, **Firebase를 연결하면 각자 기기에서 함께** 칠 수도 있습니다. 림버스 판과 같은 구조(오프라인 단독 + 선택적 온라인)예요.

---

## 파일 구성

| 파일 | 역할 |
|------|------|
| `index.html` | 화면 구조 (세팅 · 게임판) |
| `style.css` | 디자인 |
| `game.js` | **순수 룰 엔진** (덱 · 합법수 · 공격 스택 · 반격 · 특수카드) |
| `app.js` | UI · 카드 렌더 · 패스앤플레이 흐름 |
| `realtime.js` | 온라인 멀티 (Firebase, 호스트 권한) |
| `firebase-config.js` | **여기에 본인 Firebase 값을 넣으면 온라인이 켜짐** |

`firebase-config.js`를 비워두면 온라인 버튼은 비활성이고, **한 기기 패스앤플레이**로 정상 동작합니다.

---

## 기본 규칙

- 위 카드와 **무늬가 같거나 / 숫자·문자가 같은** 카드를 냅니다.
- 낼 카드가 없으면 **1장 뽑고** 차례를 넘깁니다.
- 손패를 가장 먼저 비우면 **승리**. 1장 남으면 화면에 **원카드** 표시가 뜹니다.

### 특수 카드 (세팅에서 조정 가능)

| 카드 | 효과(기본값) |
|------|------|
| **2** | 공격 — 다음 사람 **+2장** |
| **A** | 공격 — 다음 사람 **+3장** |
| **블랙 조커** | 공격 **+5장** |
| **컬러 조커** | 공격 **+7장** |
| **7** | **무늬 변경** (낼 때 무늬 선택) |
| **J** | 다음 사람 **건너뛰기** |
| **Q** | **방향 전환** |
| **K** | **한 번 더 내기** |

### 공격 · 반격(되받아치기)

공격 카드를 받으면 화면 중앙에 **받을 카드 누적 카운터**가 뜹니다.
- **공격 카드로 반격**하면 장수가 쌓여 다음 사람에게 넘어갑니다.
- 반격을 못 하면(또는 안 하면) **누적된 장수를 전부 가져옵니다**.
- 기본: **2·A는 서로 반격**으로 쌓이고, **조커는 조커끼리만** 쌓입니다. (고급 룰에서 "조커로 숫자 공격 반격 허용" 켜기 가능)

자기 차례에 하단에 **제출 가능 여부 / 반격 가능 여부**가 표시됩니다.

## 한 기기로 바로 하기 (패스앤플레이)

1. `index.html`을 브라우저로 엽니다.
2. 인원 수 · 이름 · 시작 카드 수 · 조커 허용 등을 세팅.
3. **“한 기기에서 시작”** → 차례마다 **가림막**이 떠서 다른 사람은 화면을 안 보게 한 뒤, 본인만 손패를 확인하고 카드를 냅니다.

---

## 각자 기기에서 함께 하기 (온라인, 선택)

GitHub Pages 같은 정적 호스팅에는 서버가 없어서 실시간 동기화에 외부 DB가 필요합니다. 가장 간단한 무료 옵션이 **Firebase Realtime Database**입니다. (림버스 판에서 쓰던 것과 동일)

1. <https://console.firebase.google.com> → **프로젝트 추가**
2. **빌드 → Realtime Database → 데이터베이스 만들기**
   - 위치: 아무거나 (예: `asia-southeast1`)
   - 보안 규칙: 처음엔 **“테스트 모드로 시작”**
3. 프로젝트 설정(⚙️) → **내 앱**에서 웹 앱(`</>`) 추가 → `firebaseConfig` 값 복사
4. `firebase-config.js`의 `YOUR_...`를 본인 값으로 교체. **`databaseURL`이 꼭 있어야** 온라인이 켜집니다.

### 보안 규칙 (권장)

테스트 모드는 일정 기간 후 잠깁니다. Realtime Database → **규칙** 탭에 아래를 넣으면 `rooms`만 열립니다(친구끼리 캐주얼 용도 기준):

```json
{ "rules": { "rooms": { ".read": true, ".write": true } } }
```

### 진행 방식

- **마스터**: 세팅 후 **“마스터로 방 만들기”** → 대기실에서 방 코드/초대 링크를 공유 → 모두 모이면 **게임 시작**.
- **플레이어**: 받은 **초대 링크**를 열거나, 방 코드 + 닉네임으로 **참가** → 대기실에서 기다림.
- 게임 상태(엔진)는 **마스터**가 들고 모든 수를 검증·적용합니다. 각 플레이어는 **자기 손패만** 보고, 카드는 자기 차례에만 낼 수 있습니다.

> 마스터가 나가면 진행이 멈춥니다(호스트 권한 방식). 마스터는 끝까지 접속을 유지하세요.

## GitHub Pages 배포

1. 새 저장소에 위 파일들을 모두 올립니다.
   ```bash
   git init && git add . && git commit -m "onecard"
   git branch -M main
   git remote add origin https://github.com/<아이디>/<저장소>.git
   git push -u origin main
   ```
2. 저장소 **Settings → Pages** → Source를 `main` / `/(root)`로 설정.
3. 잠시 후 `https://<아이디>.github.io/<저장소>/` 로 접속.

> `firebase-config.js`의 웹 API 키는 공개돼도 괜찮은 종류지만, 위 **보안 규칙은 꼭** 설정하세요.
